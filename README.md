## 🎯 Какую проблему решает агент

### Проблема

HR-аналитики ежедневно сталкиваются с информационной перегрузкой:

- Десятки источников новостей о рынке труда
- Англоязычные источники требуют перевода
- Нет времени на постоянный мониторинг
- Важные тренды и риски остаются незамеченными
- Реактивная работа вместо проактивной аналитики

### Решение

HR AI Агент автоматически:

| Функция | Описание |
|---------|----------|
| 🔄 **Мониторинг** | Отслеживает RSS-ленты HR-источников |
| 🌐 **Перевод** | Переводит англоязычный контент на русский через GigaChat |
| 🏷️ **Классификация** | Категоризирует контент по HR-тематикам через GigaChat |
| 📊 **Анализ трендов** | Выявляет изменения и аномалии |
| 💡 **Генерация инсайтов** | Создаёт рекомендации БЕЗ запроса пользователя (через GigaChat) |
| ⚠️ **Уведомления о рисках** | Сигнализирует о рисках в зависимости от роли и чувствительности |

## 🚀 Быстрый старт

### Требования

- **Python 3.10+** — скачать с [python.org](https://python.org) (при установке обязательно поставить галочку **Add Python to PATH**)
- **Git** — скачать с [git-scm.com](https://git-scm.com)
- **GigaChat API credentials** — см. раздел ниже

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/delovar01/hr-ai-agent.git
cd hr-ai-agent

# 2. Создать виртуальное окружение
python -m venv venv

# 3. Активировать виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 4. Установить зависимости
pip install -r requirements.txt

# 5. Создать файл с ключами
cp .env.example .env
# (на Windows если cp не работает — скопируйте файл .env.example и переименуйте в .env)

# 6. Открыть .env и вписать ваш GIGACHAT_CREDENTIALS (см. инструкцию ниже)

# 7. Запустить дашборд
streamlit run dashboard/app.py
```

После запуска откроется браузер с дашбордом.

## 🔑 Как получить GIGACHAT_CREDENTIALS

`GIGACHAT_CREDENTIALS` — это ключ авторизации для API GigaChat от Сбера. Он нужен, чтобы агент мог переводить тексты, классифицировать новости и генерировать инсайты.

### Пошаговая инструкция:

1. Перейдите на [developers.sber.ru](https://developers.sber.ru)
2. Зарегистрируйтесь или войдите через Сбер ID
3. Перейдите в раздел **GigaChat API** → **Создать проект**
4. Создайте новый проект (название любое, например "HR Agent")
5. В настройках проекта найдите **Client ID** и **Client Secret**
6. Сформируйте credentials: это строка в формате Base64 от `client_id:client_secret`
   - Можно сгенерировать на сайте Сбера в разделе проекта (кнопка "Скопировать credentials")
7. Вставьте полученную строку в файл `.env`:

```env
GIGACHAT_CREDENTIALS=ваша_строка_credentials_здесь
GIGACHAT_SCOPE=GIGACHAT_API_PERS
```

> **Примечание:** `GIGACHAT_API_PERS` — это scope для персонального использования (бесплатно). Для коммерческого использования нужен `GIGACHAT_API_CORP`.

## 🖥 Как пользоваться дашбордом

### Шаг 1: Настройте агента (левая панель)
- **Роль** — выберите вашу роль (HR-аналитик, HR-партнёр, HR-руководитель). От роли зависят приоритетные темы.
- **Временной диапазон** — за какой период анализировать новости (час, сутки, неделя, месяц).
- **Приоритетные темы** — галочками отметьте интересующие HR-темы.
- **Чувствительность к рискам** — порог срабатывания уведомлений:
  - Низкая — только критичные ситуации
  - Средняя — средний и высокий риск
  - Высокая — уведомлять обо всём

### Шаг 2: Запустите цикл анализа
Нажмите кнопку **«Запустить цикл анализа»** в верхней части дашборда. Агент:
1. Соберёт новости из 7 RSS-источников
2. Переведёт англоязычные через GigaChat
3. Классифицирует каждую новость по HR-теме
4. Рассчитает риск-скор
5. Сгенерирует инсайты (рекомендации) для значимых новостей
6. Создаст уведомления при превышении порога риска

### Шаг 3: Изучите результаты
- **Проактивные инсайты** — рекомендации от GigaChat: что произошло, почему важно, что делать
- **Уведомления о рисках** — сигналы о ситуациях, превысивших порог чувствительности
- **Тренды** — графики изменения риск-скора по каждой HR-теме

### Шаг 4: Запустите повторно
Каждый новый цикл обрабатывает только **новые** (ещё не обработанные) новости. Для полного сброса нажмите «Сбросить состояние агента» в левой панели.

## 📡 Откуда берутся данные

Агент собирает данные из двух типов источников:

### RSS-ленты (реальные новости)

| Источник | Язык | Тематика |
|----------|------|----------|
| Harvard Business Review | EN | Управление, HR-стратегия |
| SHRM News | EN | Новости от крупнейшей HR-ассоциации |
| VC.ru | RU | Российские бизнес-новости |
| Habr | RU | Технологии, IT-рынок |
| The Muse | EN | Карьера, рынок труда |
| Hacker News | EN | Технологические тренды |
| Reddit CS Careers | EN | Обсуждения IT-карьеры |

### Демо-данные

Для демонстрации работы агент также генерирует тестовые данные (помечены как mock). В production-версии их заменяют реальные HR API (HeadHunter, корпоративные системы).

> **Важно:** Новости используются как внешний сигнал. HR-аналитика в production работает с внутренними данными компании.

## 📖 Глоссарий терминов

| Термин | Что это значит |
|--------|---------------|
| **Проактивный инсайт** | Рекомендация, которую агент генерирует **сам**, без запроса пользователя. GigaChat анализирует новость и формулирует: что произошло, почему важно для HR, что делать. |
| **Уведомление о риске** | Автоматический сигнал, что риск-скор новости превысил порог чувствительности. Чем выше чувствительность — тем ниже порог и больше уведомлений. |
| **Риск-скор** | Числовая оценка от 0.0 до 1.0, рассчитывается по ключевым словам. 0.8+ = критический, 0.6+ = высокий, 0.4+ = средний, ниже = низкий. |
| **Аномалия** | Резкое отклонение текущего риск-скора от среднего (>30%). Означает что-то необычное по теме. |
| **Цикл мониторинга** | Один запуск агента: сбор новостей → перевод → классификация → расчёт риска → генерация инсайтов. |
| **Чувствительность** | Порог срабатывания уведомлений. Высокая = реагировать на всё (порог 0.3), средняя (0.5), низкая = только критичное (0.7). |

## 🤖 Почему это AI-агент, а не чат-бот

### Определение AI-агента

AI-агент — это автономная система, которая:

1. **Имеет состояние** (State) — помнит предыдущие наблюдения
2. **Следует правилам** (Rules) — применяет доменную логику
3. **Выполняет действия** (Actions) — инициативно генерирует выводы

### Сравнение

| Характеристика | Чат-бот | Поисковик | **HR AI Агент** |
|----------------|---------|-----------|-----------------|
| Инициатива | ❌ Ждёт запроса | ❌ Ждёт запроса | ✅ Проактивен |
| Память | ❌ Сессионная | ❌ Нет | ✅ Персистентная |
| Контекст времени | ❌ Нет | ❌ Нет | ✅ Историческое сравнение |
| Персонализация | 📝 Через промпты | ❌ Нет | ⚙️ Через конфигурацию |
| Выходные данные | 💬 Ответы | 🔗 Ссылки | 💡 Инсайты + ⚠️ Уведомления |
| Доменная логика | ❌ Общая | ❌ Общая | ✅ HR-специфичная |

### Почему нельзя заменить Perplexity или промптом

| Возможность | Perplexity/ChatGPT | HR AI Агент |
|-------------|--------------------| ------------|
| Запомнить, что вчера было 10 новостей о сокращениях, а сегодня — 50 | ❌ Нет памяти | ✅ Персистентное состояние |
| Автоматически проверить источники каждые 15 минут | ❌ Только по запросу | ✅ Фоновый мониторинг |
| Применить правило "если риск > 0.7 и тема = увольнения → уведомление" | ❌ Нет правил | ✅ Конфигурируемые правила |
| Адаптировать выходные данные под роль HR-аналитика vs HR-руководителя | ❌ Промпты | ✅ Профили ролей |
| Выявить аномалию: "В январе упоминаний burnout было 5%, сейчас — 25%" | ❌ Нет временного контекста | ✅ Анализ трендов |

### Формула

```
AI Agent = State + Rules + Autonomous Actions
Chatbot  = Stateless + Generic + Reactive
```

### Компоненты агента в коде

```python
# 1. СОСТОЯНИЕ (State) — src/agent/state.py
class AgentState:
    state = {
        "observations": [],      # История наблюдений
        "insights": [],          # Сгенерированные инсайты
        "trends": {},            # Данные по трендам
        "alerts": [],            # Уведомления о рисках
        "processed_urls": []     # Обработанные источники
    }

# 2. ПРАВИЛА (Rules) — src/agent/rules.py
class HRRules:
    RISK_KEYWORDS = {...}           # HR-специфичные ключевые слова
    def calculate_risk_score()      # Расчёт уровня риска
    def detect_anomaly()            # Выявление аномалий
    def should_generate_alert()     # Логика создания уведомлений

# 3. ДЕЙСТВИЯ (Actions) — src/agent/core.py
class HRAgent:
    def run_cycle():                # Автономный цикл работы
        # Проактивно:
        # - Мониторит источники
        # - Анализирует данные
        # - Генерирует инсайты
        # - Создаёт уведомления
```

### Триггеры генерации инсайтов

Агент генерирует инсайт, если выполняется хотя бы одно условие:

```python
def _should_generate_insight(self, item):
    # 1. Высокий риск (risk_score >= 0.6)
    # 2. Тема входит в приоритетные для пользователя
    # 3. Обнаружена аномалия (отклонение > 30%)
```

## 🔮 Использование GigaChat

### Где используется GigaChat

GigaChat (модель от Сбера) используется для четырёх задач:

| Задача | Метод | Описание |
|--------|-------|----------|
| Перевод | `translate_to_russian()` | Перевод англоязычного контента с сохранением HR-терминологии |
| Классификация | `classify_hr_topic()` | Определение HR-тематики (найм, сокращения, зарплаты и т.д.) |
| Генерация инсайтов | `generate_insight()` | Создание рекомендаций: что произошло, почему важно, что делать |
| Анализ трендов | `analyze_trend()` | Сравнение текущих данных с историческими |

### Пример

```python
from src.api.gigachat import gigachat_client

# Перевод
ru_text = gigachat_client.translate_to_russian(
    "Tech Giants Announce Major Layoffs"
)
# → "Технологические гиганты объявили о массовых сокращениях"

# Классификация
topic = gigachat_client.classify_hr_topic(ru_text)
# → {"topic": "layoffs", "confidence": 0.92, "keywords": ["сокращения"]}

# Генерация инсайта
insight = gigachat_client.generate_insight(data, context)
# → {
#     "what_changed": "Крупные IT-компании объявили о сокращениях...",
#     "why_important": "Это может повлиять на рынок труда...",
#     "recommendation": "Рекомендуется проанализировать..."
# }
```

## ⚠️ Ограничения и планы развития

### Текущие ограничения

| Ограничение | Описание | Планируемое решение |
|-------------|----------|---------------------|
| Mock-данные | Часть данных генерируется для демо | Подключение реальных HR API |
| Один пользователь | Нет многопользовательского режима | Добавление авторизации |
| Локальное хранилище | JSON-файл для состояния | PostgreSQL / Redis |
| Ручной запуск циклов | Требуется нажатие кнопки | Celery / cron-задачи |

### План развития

1. **v1.1** — Интеграция с реальными HR API (HeadHunter, SuperJob)
2. **v1.2** — Многопользовательский режим с ролями
3. **v1.3** — Прогнозирование трендов (ML-модели)
4. **v1.4** — Интеграция с корпоративными мессенджерами (Telegram, Slack)
5. **v2.0** — RAG-модуль для работы с внутренними HR-документами
